## 1.自动测试化的流程

自动化测试通过脚本（代码）模拟人工操作，自动化执行用例、验证系统表现、并且输出结果。目的：

- 测试效率
- 准确率
- 可重复性

实战经验总结：

1. 分析文档（理解需求）
   - **核心功能**  
     - 登录
       - 方法：post
       - url：
       - body：
         - username
         - password
     - 创建任务
        - 方法：possst
        - url：
        - body：
           - title（选填）
           - is_done（选填）
     - 查询任务列表
       - 方法：get
       - url：
       - 参数：
         - page
         - size 
     - 删除任务
        - 方法：delete
        - url：
        - 参数：
           - todo_id（必填） 

   - 次要功能
   - 辅助功能
   - 理解：
      - 有些接口有安全性要求：需要先登录再使用
      - 任务id：需要动态获取（接口关联：数据库查询，其他接口）
      - 实际工作需要所有接口全部测试

2. 设计用例
   1. 用例标题：登录的账号可以删除任务
   2. 前置条件：
      1. 已登录（得到了身份信息）
      2. 有任务（已经创建待删除的任务）
   3. 测试步骤：
      1. 得到任务id
      2. 调用删除接口（结果1）
      3. 调用查询接口（结果2）
   4. 断言（验证结果）：
      1. 结果1成功
      2. 结果2失败

3. 数据准备
   - 可登录的账号（bf@qq.com;bf123456）
   - 可测试的任务
   - 数据驱动测试的数据文件

4. 编写脚本
   1. 纯代码脚本
   2. 框架的用例
     



## 2.接口客户端requests

> 1. 不支持异步

### requests.Response属性

- 状态码
- 响应头

```python
method = 'get'
url ='http://www.baidu.com'
resp = requests.request(method,url)
print("行：",resp.status_code,resp.reason)
print("头：",resp.headers,)
print("头：",dict(resp.cookies),)
```
- 响应体

```python
print(resp.content ) #二进制：传输
print(resp.text ) #文本：阅读
print(resp.json() ) #字典：使用
```
