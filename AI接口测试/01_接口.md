# 1.接口（API）是什么？
顾客→服务员→厨师

- 顾客（前端）：点菜（发起请求Request）
- 服务员（API）：传递菜单、端菜（接受请求、返回响应Response）
- 后厨（后端）：做菜（处理业务逻辑）

接口：全程Application Programming Interface，**应用程序编程接口**

作用：给前端和后端提供交互数据的**通道**

- 接口是软件组件之间相互通信的协议和桥梁
- 前端（如浏览器）通过调用接口来获取数据或执行操作
- 后端处理请求并通过接口返回结果

```
Status:200
Type:Post
```

# 2.HTTP协议：接口测试的“世界语”
## 什么是HTTP协议
- HTTP：超文本传输协议
- Web通信的基础协议，客户端和服务器之间的**请求和响应**的标准
- 协议：双方预先约定好的通信规则

## “地址簿”：URL的组成
https://api.example.com:443/v1/users?id=1
- 协议（https）：通信协议
- 主机名（api.example.com）：服务器地址
- 端口（443）：服务器入口（通常省略）
- 路径（/v1/users）：资源位置
- 查询参数（?id=1)：过滤条件

## “动词”：请求方法
- GET:查看/获取数据
- POST：提交/创建数据
- PUT：更新/替换数据
- DELETE：删除数据
